Pattern

<input type="text" pattern="RegExp">

*If specified, the attribute's value must match the JavaScript Pattern production. [ECMA262]*

*The pattern is compiled "with the global, ignoreCase, and multiline flags disabled"*

*Per spec, will not report an error if the user submits an empty string. To avoid, use in conjunction with required.*

*Pattern attribute is ignored unless input is of type:text, search, url, tel, email, or password.*

  • HTML5: The pattern attribute specifies a regular expression against which the control's value, or, when the multiple attribute applies and is set, the control's values, are to be checked.
  • JS: Checks the pattern using the same HTML pattern. If you want/need to use modifiers, you must set an additional pattern in the XML file (or write your patterns better ;)). This is a limitation of the HTML5 spec. return (pattern.test(val) ? false : true);
  • PHP: Checks the pattern using the same HTML pattern. If you want/need to use additional PCRE syntax, you must set an additional pattern in the XML file. return (preg_match($pattern,$val) != false ? false : true);
<input type="text" pattern="[a-z]{2}">
<input type="email" pattern="[a-z]{2}">
<input type="url" pattern="[a-z]{2}">
<input type="submit" pattern="[a-z]{2}">
<input type="email" pattern="[a-z]{2}.+">
<input type="url" pattern="[a-z]{1}.+">
<input type="newtype" pattern="[a-z]{2}" required">

Multiple

<input type="email" multiple> OR <input type="email" multiple="multiple">

*Only applies to inputs: Email and File*

  • HTML5: May validate the field.
  • JS: Upon submission of the form, explode string by comma's and trim leading and trailing white space. Validate each value in loop.
  • PHP: Upon submission of this form, loop through data array/variable validating each value, trimming whitespace.
